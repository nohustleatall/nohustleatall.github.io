Build notes

Codex 0.128.0 on Orange Pi RV2

The current working build targets riscv64gc-unknown-linux-gnu for the Orange Pi RV2 / SpacemiT K1 class of boards.

What changed

Codex 0.128.0 builds with the same board-specific compatibility work used for the earlier Orange Pi RV2 release, updated against the newer upstream Codex tag.

  • Build OpenSSL from source for the RISC-V target.
  • Skip vendored bubblewrap during cross-compilation when target libcap is not available through pkg-config.
  • Select seccompiler's riscv64 target architecture for the network seccomp filter.
  • Use a locally built RISC-V rusty_v8 archive for Codex code-mode support.

Current artifact

The 0.128.0 binary is dynamically linked for RISC-V Linux and depends on the usual system C runtime libraries plus libgcc_s. On the target board, system bubblewrap is expected to provide the filesystem sandbox path.
